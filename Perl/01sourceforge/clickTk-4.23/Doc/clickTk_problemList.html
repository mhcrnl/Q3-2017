P079

Constructor ‘NoteBook(  ) ‘ cannot be successfully parsed

Description

 
It seems that the empty options list of the constructor cannot be parsed. 
In fact, if an option is given , then the parser works fine. 
See project scrolledFrame.pl
 
Related doc: problemlist\P079\P079.doc          

Reason

 
 Wrong handling of option’s list in bottomUpParser. When the list is empty , then the array @values remains empty. Thus, no reduce step for the empty list get triggered and the productions iterator get not increased. Therefore, the next token get processed with the wrong production, which of course cannot emit the token.
Bugfix: when a list has been  shifted, then a following reduce step must be triggered. Therefore a new state ‘end of option list’ must be implemented.
Note: the implemented solution is just a temporary fix …
Open items : 
- simplify parameter structure,
- Integrate command-types into parser-paramters in order to avoid parsing for each expected command type.

Modifs

 
New state for lists EOL, 
Recursive handling of option’s list

Status

P059

Let edit the option ‘Callback’ out of dialog ‘edit widget options’

Description

 
Callback codes may be conceived as an option of a widget, in example the code to implement the action of a button. Though, a callback may also be conceived simply as a call to a more general subroutine, which resides somewhere in the target code, or even in another module which is not part of the project. Thus, the latter conception requires a splitting of the input: on the one hand the action code itself and on the other hand the call. So, it is reasonable that the first part is entered as a property of the project, while the second part may be entered as an option of the widget.
