The Revision Cycle 

Collect items
	number items 
	open an item for each bug
		- description of the bug (symptoms)
		- collect documents (screenshots, logfile, projects, ...) into folder ./doc/problemlist/<problemNr>
	open an item for each upgrade suggestion
		- description

Specify the changes to be implemented

	- inspect the problem list fro open items
	- select problems to be solved depending on severity
	- select upgrades depending on compatibility and Time/costs
	- set up new release identification 
	- specify the changes
	- set up the implementation plan
	- set up compatibility 
		set up conversion rules for projects, templates
		set up installations rules

Update the system

	- implement the modifications
	- test the modifications
	- set up a list of the modified items
		module ./lib/*.pl ./lib/*.pm
		scripts ./*.pl
		executable ./*.exe
		control files 
		./widgets/*.* ./templates/*.pl ./projects/*.pl
		./toolbar
		./ctkAssistentFuncList.txt
		./ctkConfigOptions.txt
		./ctk_session_Tkadmin.txt
	- adapt doc/clickTk_problemList.html


Update examples
	
Update the documentation

	- adapt clicktk_readme.txt
	- adapt file README
	- adapt doc/clickTk_Maintenance.html
	- adapt clickTk_version_<version>_changes.html
	- adapt screen shots 
	- adapt user guide  ./doc/userd/user_doc.html (winword)
	- adapt diagrams	./doc/diagrams
	- adapt clicktk_readme.txt
	- adapt ./doc/programmers_guide.html

Set up deliverable

	-run script ctkBuild.pl
	-check deliveable with winzip (test)

Save the revision

	- create revision definition file
	  ./clickTk-<version>.txt
	- run script ctkSaveRevision.pl 

Move revision to prod environment

	- merge folders with beyondcompare2

Update homepage ./homepage/*.*

	- adapt date in hp_corner.html
	- adapt items in hp_downloads.html
	- check local homepage

Upload homepage files ( WS_FTP95 LE )

	- upload documents
	- upload pages
